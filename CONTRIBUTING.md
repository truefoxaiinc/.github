# Contributing

Thank you for helping improve Truefox AI software.

## Before you begin

- Search existing issues and pull requests before opening a new one.
- Use a security advisory or the private security contact for vulnerabilities. Never disclose a vulnerability in a public issue.
- Keep changes focused and avoid unrelated formatting or dependency updates.
- Do not commit credentials, customer data, personal data, generated files or proprietary assets.

## Development workflow

1. Fork or create a feature branch from the repository's default branch.
2. Follow the repository's local-development instructions.
3. Add or update tests and documentation.
4. Run the relevant lint, type, test and build checks.
5. Open a pull request using the provided template.

Use conventional commit prefixes such as `feat:`, `fix:`, `docs:`, `test:`, `ci:`, `security:` and `chore:`.

By contributing, you agree to follow the Code of Conduct and confirm that you have the right to submit the work under the repository's license.
