# Implementation roadmap

## Phase 1 — Immediate

- Publish the company profile and fox-only symbol.
- Publish verified website, contact, LinkedIn, careers, security and support channels.
- Set account metadata and a professional social preview.
- Leave project pins empty until publication-ready repositories exist.

## Phase 2 — Repository quality

- Review the visibility of `adhd-assessment-platform` immediately.
- Remove user media and unnecessary generated artifacts from current and historical commits.
- Decide proprietary or approved open-source licensing.
- Add architecture, ownership, support and complete README documentation.
- Review dependencies and separate runtime from development packages.

## Phase 3 — Automation

- Add language-appropriate lint, test, build and container checks.
- Enable dependency review, Dependabot, CodeQL and secret scanning.
- Require successful checks on protected branches.
- Introduce semantic releases, release notes and artifact provenance.

## Phase 4 — Enterprise readiness

- Document privacy controls, threat models, architecture decisions and data retention.
- Establish incident response, audit logging and access-review procedures.
- Define repository creation, ownership, review and publication governance.
- Reassess conversion to a GitHub organization when a separate personal owner account is available.
